# COPPE — checklist antes de una reunión

## El día anterior

- [ ] Confirmar sector, tamaño y canales de la empresa.
- [ ] Personalizar las preguntas de descubrimiento.
- [ ] Sustituir `[EMPRESA CLIENTE]` y `[FECHA]` en la propuesta.
- [ ] Ejecutar `npm run demo:seed`.
- [ ] Ejecutar `npm run demo:verify`.
- [ ] Probar el guion completo.
- [ ] Comprobar cargador, conexión y pantalla compartida.
- [ ] Tener disponible una conexión alternativa.

## Treinta minutos antes

- [ ] Cerrar notificaciones y aplicaciones personales.
- [ ] Ejecutar `npm run dev`.
- [ ] Ejecutar `npm run demo:login`.
- [ ] Abrir `http://127.0.0.1:3417`.
- [ ] Confirmar que aparece `Hotel Costa Azul — Demo COPPE`.
- [ ] Abrir el caso de David Martín.
- [ ] Volver al dashboard.
- [ ] Abrir la presentación.
- [ ] Compartir solo la ventana necesaria.

## Durante

- [ ] Preguntar antes de mostrar funcionalidades.
- [ ] Usar únicamente datos ficticios.
- [ ] No pulsar envíos de proveedores.
- [ ] No abrir paneles, terminales ni secretos.
- [ ] No afirmar certificaciones o SLA inexistentes.
- [ ] Registrar necesidades y objeciones fuera de COPPE.
- [ ] Cerrar con siguiente paso y fecha.

## Después

- [ ] Enviar resumen y próximos pasos.
- [ ] Personalizar alcance y métricas.
- [ ] No solicitar datos reales todavía.
- [ ] Si existe aceptación, iniciar constitución y preparación de producción.
- [ ] Convertir la aceptación en pedido condicionado.
